# 🧪 卜日记  |  用经济学回归，算一算“算命”到底准不准

> 一个面向实证研究的开源多算法命理工具集，帮你用数据验证“算命到底准不准”。

⚠️ **重要声明**：本项目仅为社会观察类实证研究工具，所有内容均作娱乐与学术参考，不宣扬封建迷信，不构成任何人生决策建议。所有术数计算均调用开源算法库，无任何人工/AI编造内容。
---

## 👤 我的出发点：一个经济学学生的“唯物主义”困惑
> “如果玄学无法被数据证伪，那它或许就是另一种形式的统计学。”

作为经济学出身的学生，长期的专业训练塑造了我认识世界的新方式：万物皆可量化，一切现象背后都隐藏着变量之间的相关系数（r）。我习惯用回归分析拆解世界，始终相信只要样本量足够大，噪声终会被过滤，真实的信号一定会浮现。但“算命”始终是我知识体系里的一个异常值。

在我看来，传统命理学的本质，其实是古人在缺乏计算工具的年代，通过口传心授汇集海量样本后沉淀的经验主义概率推论。但现实里的矛盾始终无解：
同一件事，八字断“今日破财”，西洋占星却显示“金星入庙财运亨通”；
同一段感情，紫微斗数说“姻缘将至”，塔罗牌却抽出“高塔”预示关系破裂。

没有统一的度量衡，没有客观的验证标准，用户最终只能陷入“不可知论”：要么盲信那个“听起来更顺耳”的结论，要么困在“信则灵”的心理暗示里，所有流传的“准”，本质上都只是幸存者偏差。

我启动这个项目的核心目的很简单：搞清楚到底哪一种预测方法更准？还是说，它们都不准？

## 🎯 覆盖所有立场的无偏实验
本项目从设计之初就不预设“信/不信玄学”的立场，无论你属于哪一类，都能找到参与价值：
- 如果你是玄学爱好者：不用再纠结“不同方法算出来结果不一样该信谁”，你可以一键用多流派工具测算同一件事，后续也能直接参考全量数据的命中率结论，选择对应领域最可靠的测算方式；
- 如果你是中立观望者：可以纯当“玄学观测者”，既可以匿名贡献自己的真实生活数据，也能随时查看公开的回归看板，直观看到各类玄学方法和真实世界的重合度，不用自己踩坑就能拿到客观结论；
- 如果你和我一样是实证派：我们会完整公开统计维度、计算逻辑和原始匿名样本，陪你一起验证中西方不同玄学方法的预测结果和真实事件的重合度——最终要么评选出各细分领域最准确的测算方式，要么用足量数据完成对玄学的证伪。

## ⚖️ 证伪，或是重构
作为坚定的唯物主义者，我最初在最初设计产品时甚至带有强烈的“证伪”目的，我预设了两种完全不同的实验结果：
- **假设A（唯物主义结算时刻）：证伪**
当我们收集到足够多用户的真实生活数据，和各类流派的预测结果做回归分析后，如果发现所有玄学预测和实际事件的相关系数都趋近于0，那就可以用科学方法证明玄学只是随机噪声或心理安慰，对厘清认知、破除迷信有明确的价值。
- **假设B（迷人的可能性）：发现规律**
如果在大样本下，我们意外发现某些算法在特定维度（比如健康、情绪、特定行业运势）上，和实际数据呈现出统计学显著的正相关（$p < 0.05$），那项目的意义就会变成“筛选器”：我们会帮大众从纷繁的流派里找出真正有预测效力的方法，并且明确它的适用边界——比如“八字看宏观流年更准，塔罗在微观心理投射上更有效”，让玄学从“玄之又玄”变成可量化、可参考的概率工具。

我不希望这个平台成为又一个算命工具，我想把它做成玄学界的双盲实验平台：让数据说话，而不是让大师说话；让回归分析决定哪种方法值得参考，而不是靠口碑或传说，做成玄学界的“循证医学”。

## 🔬 当前产品阶段的样本收集逻辑
彻底规避“主观解读”“信则灵”的干扰，所有环节标准化、可追溯：

### 「预测端」统一结构化入库
一方面接入八字、紫微、占星等主流标准化测算工具，用户可以一键完成多流派同主题测算；另一方面全网爬取主流命理博主公开发布的日/月/年运势内容，按流派、预测时间、预测维度（事业/姻缘/健康/情绪/财运等）做结构化拆分存储，消除模糊表述的干扰。

### 「事实端」盲测采集真实数据
接入豆包大模型，基于各流派的预测内容生成完全不关联任何流派的中性确认选择题，比如“今日是否出现以下不适：肾/肝/口鼻/眼睛”“今日是否有计划外的收入/支出”，用户仅需根据真实情况作答，全程不知道问题对应哪种测算方法，从根源上规避“希望某方法准”的心理暗示偏差。

### 「校验端」大模型自动打分
由大模型对「结构化预测内容」和「用户真实回答」做语义匹配，计算二者diff，差异越小则该条预测的命中率得分越高，全程无人工干预，彻底解决不同人对“准”的判断标准不一致的问题。

### 「输出端」公开回归看板
积累足够样本后，所有匿名数据会汇总为公开的全量回归看板，既展示所有流派的整体命中率排名，也会细分到不同维度做区分——比如明确给出“八字在事业流年上命中率72%，在健康问题上命中率41%”“塔罗在短期情绪预判上命中率68%”这类具象可参考的结论。

## 🚀 之后的能力迭代规划
从小到大逐步推进，先跑通核心逻辑再拓展边界：

## ✨ 核心特性
### 🎯 设计理念
- **无偏验证**：所有测算结果、记录、统计全程无人工干预，完全基于开源算法和用户真实记录计算准确率、Pearson相关系数，杜绝“信则灵”的主观偏差；
- **隐私第一**：出生信息、日记记录、收藏内容100%存储在用户浏览器本地（localStorage + IndexedDB）；
- **完全开源**：所有算法逻辑、统计规则透明可查，没有黑箱，所有人都可以基于源码做二次研究。

### 🔧 技术亮点
- **可信计算**：所有术数算法均调用行业通用开源库，没有任何LLM生成的虚假结果，测算逻辑可追溯；
- **零后端纯静态**：打包后可直接部署在任意静态托管平台（Cloudflare Pages/Vercel/GitHub Pages等），无需服务器成本；

## 📱 功能模块
| 路径 | 模块 | 功能说明 | 数据存储 |
|---|---|---|---|
| `/` | 首页/出生档案 | 录入基础出生信息，作为所有测算的基准 | localStorage |
| `/reading` | 命盘大厅 | 支持八字、紫微斗数、西洋占星3种主流命盘一键切换查看 | 调用`taibu-core`实时本地计算 |
| `/daily` | 今日运势聚合 | 统一展示4种算法的当日各维度吉凶得分，无需逐个切换平台查询 | `taibu-core`本地计算 |
| `/diary` | 事实记录日记 | 记录当日真实发生的事件、感受，同时自动快照当日所有算法的预测结果，作为验证的“锚点”数据 | Dexie / IndexedDB 本地存储 |
| `/dashboard` | 准确率看板 | 自动统计你录入的所有数据，生成各算法的命中率、Pearson相关系数，细分事业/姻缘/健康/财运等维度 | 本地离线统计 |
| `/influencers` | 命理博主收藏 | 支持收藏各平台命理博主的主页/RSS链接，方便统一查看不同博主的运势观点，避免跨平台跳转 | Dexie / IndexedDB 本地存储 |

## 🔬 核心算法依赖
所有测算均基于成熟开源术数库实现，无自研黑箱算法：
- 基础农历转换：`lunar-javascript` / `tyme4ts`
- 紫微斗数：`iztro`
- 西洋占星：`circular-natal-horoscope-js`
- 统一封装调用：`taibu-core`

## 🚀 部署教程
本项目为纯静态应用，所有托管平台均可部署，以下为推荐方案：
### 方案1：Cloudflare Pages（推荐，国内访问速度最优）
1. 将项目代码推到你的GitHub/GitLab/Bitbucket仓库
2. 登录 [Cloudflare Pages](https://pages.cloudflare.com/) → 新建项目 → 关联你的Git仓库
3. 构建配置填写：
   - 框架预设：`Vite`
   - 构建命令：`npm run build`
   - 构建输出目录：`dist`
   - 环境变量添加：`NODE_VERSION = 20`（或22）
4. 点击部署，完成后会自动分配`xxx.pages.dev`域名，可绑定自定义域名

> 不想用Git也可以本地执行`npm run build`后，用`npx wrangler pages deploy dist`一键上传部署。

### 方案2：Vercel
关联仓库 → 框架选择`Vite` → 一键部署即可，无需额外配置。

### 方案3：GitHub Pages
1. 在`vite.config.ts`中添加`base: "/xuanxue-app/"`（替换为你的仓库名）
2. 本地构建后将`dist/`目录内容推到仓库的`gh-pages`分支即可。

## 📌 已知限制
- 占星模块依赖的`circular-natal-horoscope-js`存在部分Node内置模块兼容问题，构建时会出现externalize警告，已做容错处理，不影响其他模块运行，后续将切换为`swisseph-wasm`提升占星精度；
- 当前打包后体积约7MB（gzip后2.4MB），主要为紫微、占星算法库体积，后续将通过动态导入实现按需加载优化；
